Visual Studio

C/C++ 프로그램을 작성하기 위해서는 편집기가 필요하며, 이렇게 작성한 C 프로그램을 실행하기 위해서는 컴파일러와 링커 등이 필요합니다. Visual Studio는 이런 도구들을 모두 제공해주는 IDE(Integrated Development Environment; 통합 개발 환경)입니다. Microsoft Windows에서는 Visual Studio만 있으면 C/C++ 프로그램을 작성하고 실행할 수 있습니다.

Visual Studio 설치 프로그램 다운로드

웹 브라우저로 Visual Studio 웹사이트에 접속해서, 화면 왼쪽에 보이는 "Visual Studio 다운로드" 버튼을 클릭해 Visual Studio 2022 Community 버전을 다운로드합니다.

Visual Studio 설치

다운로드한 설치 프로그램을 실행해서 Visual Studio를 설치합니다.

Visual Studio는 Microsoft에서 제공하는 강력한 개발 도구입니다. 우리는 Visual Studio를 이용해서 다양한 프로그램을 작성할 수 있습니다. 그런데 너무 많은 기능을 제공하기 때문에 Visual Studio에서 제공하는 기능을 모두 설치하면 설치하는 데 시간도 오래 걸리고 디스크 용량도 많이 차지합니다. 그래서 당장 필요한 기능만 선택해서 설치하는데, 이렇게 필요한 기능들을 묶어놓은 단위를 Visual Studio에서는 "워크로드(workload)"라고 합니다.

우리가 C/C++ 프로그래밍을 공부하기 위해 필요한 기능은 그렇게 많지 않습니다. C/C++ 프로그래밍을 공부하면서 우리가 만들 프로그램은 모두 데스크톱 애플리케이션이므로, 위 그림처럼 "C++을 사용한 데스크톱 개발"이라는 워크로드만 설치하도록 합니다.

Tip

나중에 다른 워크로드가 필요하면, 언제든지 "Visual Studio Installer"를 실행해서 워크로드를 추가할 수 있습니다. 하지만 워크로드를 많이 추가하면 Visual Studio를 설치하는 데 오래 걸리므로 꼭 필요한 워크로드만 설치하는 게 좋습니다.

워크로드를 선택하고 "설치" 버튼을 누르면 아래 그림처럼 Visual Studio가 설치됩니다.

Visual Studio Installer는 필요한 파일을 다운로드하며 Visual Studio를 설치하기 때문에 네트워크 환경이나 컴퓨터 환경에 따라 설치 시간이 달라집니다. 조금 기다리면 설치가 끝나고 아래와 같은 화면이 표시됩니다.

이 화면은 Visual Studio에 Microsoft 계정으로 로그인하는 화면인데, 나중에 로그인할 수 있으니 "지금은 건너뛰기" 항목을 클릭해서 넘어갑니다.

"Visual Studio 환경 개인 설정" 화면에서 개발 환경을 설정합니다.

기본 개발 환경을 "Visual C++"로 선택합니다.

색 테마는 기본 값인 "어둡게"로 두고 "Visual Studio 시작" 버튼을 누릅니다.

Visual Studio를 실행하면 위 그림처럼 기존 프로젝트를 선택하거나 새로운 프로젝트를 만드는 화면이 표시됩니다. 프로젝트는 잠시 후에 만들기로 하고, "코드를 사용하지 않고 계속" 항목을 클릭합니다.

이 화면이 Visual Studio의 작업 화면입니다. 개발 환경은 잠시 후에 설정하고, 일단 Microsoft 계정으로 Visual Studio에 로그인합니다.

  1. Visual Studio 화면 상단에 있는 메뉴 바에서 "도움말" 메뉴 항목을 선택합니다.
  2. 하위 메뉴가 표시되면 "Visual Studio 등록" 메뉴 항목을 선택합니다.

Visual Studio 2022 Community 버전은 Microsoft 계정으로 로그인해서 등록하면 계속 무료로 사용할 수 있습니다. Microsoft 계정이 있으면 "로그인" 버튼을 눌러 로그인을 하고, Microsoft 계정이 없으면 "새로 만드세요" 항목을 클릭해서 화면에서 안내하는 대로 Microsoft 계정을 만들어 로그인합니다.

개발 환경 설정

이제 Visual Studio로 C/C++ 프로그램을 작성하고 실행하기 위해 개발 환경을 설정합니다. 개발 환경을 설정하는 순서는 아래와 같습니다.

  1. 작업 디렉터리 생성
  2. Visual Studio에서 새 프로젝트 생성
  3. C 프로그램 작성
  4. C 프로그램 실행

작업 디렉터리 생성

작업 디렉터리(Workspace)는 우리가 작성한 C/C++ 소스 파일을 저장하는 디렉터리(directory)를 말합니다. 작업 영역 또는 작업 폴더라고 하기도 합니다.

Visual Studio 프로젝트는 작업 디렉터리 안에 만듭니다. 정확히는, 작업 디렉터리 안에 솔루션을 만들고, 솔루션 안에 프로젝트가 포함됩니다. 프로젝트에는 C/C++ 소스 파일과 헤더 파일을 작성합니다.

Visual Studio에서는 프로젝트에 포함된 C/C++ 소스 파일들과 헤더 파일들을 이용해 프로그램 하나를 만듭니다. 솔루션에는 이러한 프로젝트들이 여러 개 포함될 수 있습니다. Visual Studio에서 솔루션과 프로젝트를 만들면 새 폴더가 추가됩니다. 이 폴더 안에는 솔루션과 프로젝트 설정 파일이 추가됩니다.

Microsoft Windows를 사용하는 경우, 파일 탐색기(File Explorer)를 열어 D 드라이브나 C 드라이브에 workspace 폴더를 만듭니다. 그리고 그 안에 clang 폴더를 만듭니다. 그러니까 작업 디렉터리의 경로는 D:\workspace\clang 또는 C:\workspace\clang이 됩니다.

작업 디렉터리는 시스템 드라이브(C 드라이브)가 아닌, 다른 드라이브에 만드는 것이 좋습니다. 컴퓨터에 C 드라이브만 있다면 어쩔 수 없겠지만, C 드라이브 외에 다른 드라이브가 있다면 그 드라이브에 작업 디렉터리를 만듭니다. 비대면으로 수업을 듣는 수강생은 USB 메모리나 외장 HDD 등을 가져와서, 거기에 작업 디렉터리를 만들도록 합니다.

Tip

폴더나 파일의 이름은 한글이나 빈칸을 쓰지 않고, 영문자와 숫자, 하이픈(-), 밑줄 문자(_)로만 작성하는 게 좋습니다.

사용하는 컴퓨터의 운영 체제가 Microsoft Windows라면, 기본적으로 파일 확장자가 보이지 않도록 설정되어 있으니 파일 확장자가 보이도록 설정하는 게 좋습니다.

아래 그림과 같이 파일 탐색기의 "보기" 메뉴에서 "파일 확장명" 항목을 체크하면 파일 확장자가 표시됩니다.

Tip

파일 확장자(Filename extension)는 파일의 이름에서 파일의 종류와 그 역할을 표시하기 위해 사용하는 부분입니다. 간단하게 확장자라고도 합니다. 대부분의 운영 체제에서는 파일 이름에서 마지막 점(.) 뒤에 나오는 부분을 확장자로 인식합니다.

C 소스 파일의 파일 확장자는 .c이고, C++ 소스 파일의 파일 확장자는 .cpp입니다.

Visual Studio에서 새 프로젝트 생성

Visual Studio에서 프로그램을 만들기 위해서는 프로젝트부터 만들어야 합니다. Visual Studio에서 프로젝트는 프로그램을 만들기 위한 단위입니다. 프로젝트에는 프로그램을 만들기 위한 여러 C/C++ 소스 파일들과 헤더 파일들이 포함됩니다. 이러한 프로젝트는 솔루션에 포함됩니다. 솔루션 안에는 여러 프로젝트가 포함될 수 있습니다. 즉, 솔루션은 프로젝트들의 묶음, 프로그램들의 묶음이라고 볼 수 있습니다.

  1. Visual Studio 화면 상단에 있는 메뉴 바에서 "파일" 메뉴 항목을 선택합니다.
  2. 하위 메뉴에서 "새로 만들기" 메뉴 항목을 선택합니다.
  3. 하위 메뉴가 표시되면 "프로젝트" 메뉴 항목을 선택합니다.

"새 프로젝트 만들기" 화면에서 새 프로젝트의 템플릿을 선택합니다. 프로젝트 템플릿은 어떤 프로그램을 만들 것인지에 따라 달라집니다.

  1. 우리는 아무런 기능도 포함되지 않은 간단한 콘솔 프로그램을 만들 것이니 "빈 프로젝트" 템플릿을 선택합니다.
  2. 프로젝트 템플릿을 선택하고 나서 "다음" 버튼을 클릭합니다.

"새 프로젝트 구성" 화면에서 프로젝트의 이름과 저장 위치, 솔루션 이름을 지정합니다. 솔루션과 프로젝트는 폴더로 생성됩니다.

  1. 프로젝트의 이름을 clang-01과 같이 입력합니다.
  2. 프로젝트 저장 위치는 "찾기" 버튼을 눌러 앞에서 만들었던 작업 디렉터리(workspace)를 선택합니다.
  3. 솔루션 이름은 clang-study과 같이 입력합니다.
  4. 제대로 입력했는지 확인하고, 문제가 없으면 "만들기" 버튼을 클릭해서 새 프로젝트를 만듭니다.

프로젝트 이름과 솔루션 이름은 위에서 말한 것처럼 지정하지 않아도 됩니다. 원하는 이름으로 지정하되, 한글이나 빈칸 등이 들어가지 않는 게 좋습니다.

프로젝트 이름을 clang-01로 지정하면, 이 프로젝트에서 빌드하면 clang-01.exe라는 실행 파일이 만들어집니다. 이걸 고려해서 프로젝트 이름을 결정합니다.

새 프로젝트를 만들면 위 그림처럼 솔루션 탐색기 창에 솔루션과 프로젝트가 표시됩니다.

Warning

Visual Studio 프로젝트에는 라이브러리나 소스 파일의 경로 정보도 포함되어 있기 때문에 다른 컴퓨터에서 만든 프로젝트를 열면 프로젝트 설정이 헝클어져서 프로젝트를 열 수 없게 됩니다. 따라서 다른 컴퓨터에서 만든 Visual Studio 프로젝트는 열지 않아야 합니다. Visual Studio에서는 학원 컴퓨터에서 사용하는 프로젝트와 개인 PC에서 사용하는 프로젝트를 따로 만들어서 사용하는 게 좋습니다.

C 프로그램 작성

새 프로젝트를 추가했으니 프로그램을 만들기 위해 C 소스 파일을 추가합니다.

수업 시간에는 시간마다 소스 파일을 여러 개 작성합니다. 그래서 시간이 지나면 소스 파일이 너무 많아져서 관리하는 게 번거롭습니다. 소스 파일들을 쉽게 관리하기 위해 수업 시간마다 폴더를 만들고, 폴더 안에 소스 파일을 작성합니다.

Visual Studio 프로젝트에서 폴더를 만들기 위해서는 솔루션 탐색기 상단에 있는 "모든 파일 표시" 버튼을 체크된 상태로 만들어야 합니다. 이 버튼이 체크된 상태가 아니면 폴더를 만들 수 없습니다.

위 그림에 나온 것처럼 "모든 파일 표시" 버튼을 찾아 클릭합니다. 그러면 솔루션 탐색기 창에서 "리소스 파일", "소스 파일", "헤더 파일" 등의 폴더가 사라집니다. 이 폴더들은 논리적인 폴더라서 솔루션 탐색기에서 보이지 않는다고 해서 실제 폴더가 지워진 것은 아닙니다.

  1. 솔루션 탐색기에서 프로젝트를 선택하고 나서, 마우스 오른쪽 버튼을 눌러 콘텍스트 메뉴를 엽니다.
  2. 콘텍스트 메뉴에서 "추가" 메뉴 항목을 선택합니다.
  3. 하위 메뉴에서 "새 폴더" 메뉴 항목을 선택합니다.

"새 폴더" 메뉴 항목은 솔루션 탐색기의 "모든 파일 표시" 버튼을 체크해야 표시됩니다.

솔루션 탐색기에서 새 폴더를 추가하면 새 폴더의 이름을 입력할 수 있습니다. 새 폴더의 이름은 day01과 같이 입력합니다.

폴더를 추가하고 나서 C 소스 파일을 추가합니다.

  1. 솔루션 탐색기에서 새로 추가한 폴더를 선택합니다. 그런 다음, 마우스 오른쪽 버튼을 눌러 콘텍스트 메뉴를 엽니다.
  2. 콘텍스트 메뉴에서 "추가" 메뉴 항목을 선택합니다.
  3. 하위 메뉴가 열리면 "새 항목" 메뉴 항목을 선택합니다.

"새 항목"을 추가했을 때, 위 그림처럼 대화 창이 표시되면 대화 창 아래에 있는 "모든 템플릿 표시" 버튼을 클릭합니다.

  1. 새 항목의 템플릿으로 "C++ 파일(.cpp)"을 선택합니다.
  2. 새 항목의 이름은 hello.c로 입력합니다. 지금은 C 소스 파일을 추가하는 것이므로 파일 확장자가 .c이어야 합니다. C++ 소스 파일은 파일 확장자가 .cpp입니다.
  3. "추가" 버튼을 클릭해 새 소스 파일을 추가합니다.

새 소스 파일을 추가하면, 파일 편집 창이 열리고 이곳에서 소스 파일을 편집할 수 있습니다. 솔루션 탐색기에는 새로 추가한 소스 파일이 표시됩니다.

파일 편집 창에 아래 내용을 입력합니다. 이때, 아래 사항을 주의합니다.

  1. 소스 코드에서 대문자와 소문자를 구분해서 입력합니다.
  2. 줄 바꿈과 띄어쓰기에 신경 써서 작성합니다.
  3. 줄 앞에 있는 공백은 탭(Tab) 키를 입력합니다.
  4. C/C++ 프로그램에서 기호들은 정해진 역할이 있으니 규칙에 맞게 사용해야 합니다.
  5. \(역슬래시; 백슬래시; Backslash) 기호는 키보드에서 백스페이스(Backspace) 키 왼쪽이나 아래에 있는 원화 기호(₩) 키를 누르면 입력됩니다. 두 기호는 같은 글자인데 글꼴이 한글 글꼴일 때는 원화 기호(₩)로 표시되고, 영문 글꼴일 때는 역슬래시(\)로 표시됩니다.
#include <stdio.h>

int main()
{
    printf("Hello World!\n");

    return 0;
}

소스 파일을 수정하면, 편집 창 상단의 탭에 표시된 파일 이름 옆에 * 기호가 추가됩니다. 이 기호가 표시된 파일은 수정하고 아직 저장하지 않은 상태입니다. 파일을 수정하면 꼭 Ctrl+S 단축키를 눌러 저장하도록 합니다.

C 프로그램 실행

C 소스 파일을 작성하고 나서 실행하려면 컴파일(Compile)하고 링크(Link)해서 실행 파일을 만들어야 합니다. 이 과정을 합해서 "빌드(Build)"라고 합니다.

빌드할 때는 Visual Studio 화면 상단의 메뉴 바에서 "빌드" 메뉴 항목을 선택하고, 그 하위 메뉴에서 "솔루션 빌드" 메뉴 항목이나 "프로젝트 빌드" 메뉴 항목을 선택하면 됩니다. 이렇게 솔루션이나 프로젝트를 빌드하면 현재 프로젝트에 포함된 소스 파일들로 실행 파일을 만듭니다.

그런데 이렇게 하면 실행 파일이 저장된 위치를 찾아가 실행해야 하기 때문에 불편합니다. 다행히 Visual Studio에서는 프로젝트를 빌드한 다음, 바로 프로그램을 실행할 수 있는 방법이 있습니다.

  1. Visual Studio 화면 상단에 있는 메뉴 바에서 "디버그" 메뉴 항목을 선택합니다.
  2. 하위 메뉴에서 "디버그하지 않고 시작" 메뉴 항목을 선택합니다.

"디버그하지 않고 시작" 기능을 사용하면 프로젝트를 빌드하고 나서 디버그 콘솔에서 프로그램을 실행해줍니다. 앞으로 많이 쓰게 될 기능이니 단축키를 외워두도록 합니다. 단축키는 Ctrl+F5입니다.

Visual Studio에서 프로젝트를 빌드하면 편집 창 아래에 출력 창이 표시되고, 이곳에 빌드 결과가 출력됩니다. 소스 파일에 오류가 있으면 이 출력 창에 오류 메시지가 표시됩니다. 오류 메시지를 잘 읽어보면 문제를 해결할 수 있습니다.

C 프로그램이 실행된 결과는 위 그림처럼 디버그 콘솔에 출력됩니다. 이 실행 결과를 보면 우리가 작성한 프로그램이 제대로 실행됐는지 확인할 수 있습니다.
