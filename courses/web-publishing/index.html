수업 중에 사용하는 프로그램

수업 중에는 아래에 나온 프로그램들을 사용합니다.

  • Visual Studio Code
  • Google Chrome

Visual Studio Code

HTML 문서를 작성하기 위해 편집기가 필요합니다. 수업 시간에는 Visual Studio Code를 사용하지만, 다른 편집기를 사용해도 상관없습니다.

Visual Studio Code 다운로드

웹 브라우저로 Visual Studio Code 웹사이트에 접속해서, 화면 왼쪽에 보이는 "Download" 버튼을 눌러 설치 파일을 다운로드합니다.

컴퓨터의 운영 체제에 따라 설치 파일이 다릅니다. Visual Studio Code 웹사이트에 접속하면 사용하는 컴퓨터의 운영 체제에 따라 설치 파일이 자동으로 결정됩니다. 다른 운영 체제의 설치 파일이 필요하면, 다운로드 페이지에서 원하는 운영 체제의 설치 파일을 다운로드할 수 있습니다.

Visual Studio Code 설치

다운로드한 설치 파일을 실행해서 Visual Studio Code를 설치합니다. Visual Studio Code를 설치하면서 따로 설정할 것은 없습니다. 다만, 아래 그림과 같이 "추가 작업 선택" 화면에서 모든 항목을 체크하고 설치하는 게 좋습니다.

Visual Studio Code 실행

위 그림과 같이 체크하고 설치했다면, 설치가 끝나고 바탕 화면에 Visual Studio Code의 바로 가기가 만들어집니다. 이 바로 가기를 더블 클릭해서 Visual Studio Code를 실행합니다.

Google Chrome

HTML 문서를 작성하면 웹 브라우저로 확인해야 합니다. 수업 시간에는 Google Chrome을 사용하지만, 다른 웹 브라우저를 사용해도 상관없습니다.

  1. 웹 브라우저로 Google Chrome 페이지에 접속해서, 설치 파일을 다운로드합니다.
  2. 다운로드한 설치 파일을 실행해 Google Chrome을 설치합니다.
    • Google Chrome을 설치하면서 따로 설정할 것은 없습니다.
  3. 설치가 끝나면 바탕 화면에 Google Chrome 바로 가기가 만들어집니다. 이 바로 가기를 더블 클릭해서 Google Chrome을 실행합니다.

개발 환경 설정

이제 Visual Studio Code로 HTML 문서를 작성하고 Google Chrome으로 HTML 문서를 확인하기 위해 개발 환경을 설정합니다. 개발 환경을 설정하는 순서는 아래와 같습니다.

  1. 작업 디렉터리 생성
  2. 작업 디렉터리에서 Visual Studio Code 실행
  3. Visual Studio Code에서 "open in browser" 확장 기능 설치
  4. 새로운 HTML 파일 생성
  5. HTML 문서 편집
  6. Google Chrome으로 작성한 HTML 문서 확인

작업 디렉터리 생성

작업 디렉터리(Workspace)는 우리가 작성한 HTML 파일 등을 저장하는 디렉터리(directory)를 말합니다. 작업 영역 또는 작업 폴더라고 하기도 합니다.

Microsoft Windows를 사용하는 경우, 파일 탐색기(File Explorer)를 열어 D 드라이브나 C 드라이브에 workspace 폴더를 만듭니다. 그리고 그 안에 html 폴더를 만듭니다. 그러니까 작업 디렉터리의 경로는 D:\workspace\html 또는 C:\workspace\html이 됩니다.

작업 디렉터리는 시스템 드라이브(C 드라이브)가 아닌, 다른 드라이브에 만드는 것이 좋습니다. 컴퓨터에 C 드라이브만 있다면 어쩔 수 없겠지만, C 드라이브 외에 다른 드라이브가 있다면 그 드라이브에 작업 디렉터리를 만듭니다. 비대면으로 수업을 듣는 수강생은 USB 메모리나 외장 HDD 등을 가져와서, 거기에 작업 디렉터리를 만들도록 합니다.

Tip

폴더나 파일의 이름은 한글이나 빈칸을 쓰지 않고, 영문자와 숫자, 하이픈(-), 밑줄 문자(_)로만 작성하는 게 좋습니다. 우리가 작성한 HTML 파일 등은 나중에 서버에 업로드해서 서비스합니다. 서버로 사용하는 컴퓨터와 우리가 작업하는 컴퓨터의 운영 체제(Operating System; OS)가 다르면, 운영 체제에서 사용하는 문자 인코딩(character encoding)도 달라집니다. 그래서 파일 이름에 영문자나 숫자가 아닌 다른 문자가 포함되면 파일 이름이 제대로 표시되지 않을 수도 있습니다.

사용하는 컴퓨터의 운영 체제가 Microsoft Windows라면, 기본적으로 파일 확장자가 보이지 않도록 설정되어 있으니 파일 확장자가 보이도록 설정하는 게 좋습니다.

아래 그림과 같이 파일 탐색기의 "보기" 메뉴에서 "파일 확장명" 항목을 체크하면 파일 확장자가 표시됩니다.

Tip

파일 확장자(Filename extension)는 파일의 이름에서 파일의 종류와 그 역할을 표시하기 위해 사용하는 부분입니다. 간단하게 확장자라고도 합니다. 대부분의 운영 체제에서는 파일 이름에서 마지막 점(.) 뒤에 나오는 부분을 확장자로 인식합니다.

작업 디렉터리에서 Visual Studio Code 실행

방금 만든 작업 디렉터리를 Visual Studio Code에서 열 수 있습니다. Visual Studio Code에서는 HTML 파일을 따로 작성하는 것보다는 이렇게 작업 디렉터리에서 작업하는 것이 더 편합니다.

Visual Studio Code에서 작업 디렉터리를 여는 방법은 여러 가지가 있습니다. 그 중에서 아래 두 가지 방법을 주로 사용합니다.

  • 파일 탐색기에서 컨텍스트 메뉴로 작업 디렉터리를 Visual Studio Code에서 열기
  • Visual Studio Code에서 "File" 메뉴의 "Open Folder" 메뉴 항목으로 작업 디렉터리 열기

위에서 작업 디렉터리를 만들면서 실행했던 파일 탐색기가 아직 실행되어 있다면, 파일 탐색기에서 작업 디렉터리로 들어가, 마우스 오른쪽 버튼을 클릭해서 컨텍스트 메뉴를 엽니다. 컨텍스트 메뉴의 내용은 컴퓨터에 따라 다를 수 있습니다. Visual Studio Code를 설치하면서 "'Code(으)로 열기' 작업을 Windows 탐색기 디렉터리의 상황에 맞는 메뉴에 추가" 항목을 체크했다면, 아래 그림과 같이 "Code(으)로 열기"라는 메뉴 항목이 나옵니다.

Visual Studio Code에서 "File" 메뉴의 "Open Folder" 메뉴 항목으로 작업 디렉터리를 선택해서 작업 디렉터리를 열 수도 있습니다.

Visual Studio Code에서 "open in browser" 확장 기능 설치

HTML 문서를 작성한 다음에는 제대로 작성했는지 웹 브라우저로 확인해봐야 합니다. HTML 문서를 웹 브라우저로 확인하려면, 파일 탐색기를 열어 확인하고자 하는 HTML 파일을 선택해서 웹 브라우저로 확인합니다. 이런 과정이 번거롭기 때문에 Visual Studio Code에서 "open in browser" 확장 기능을 설치합니다.

  1. Visual Studio Code 화면 왼쪽에 있는 작업 막대(Activity Bar)에서 확장 기능(Extensions) 버튼을 클릭합니다.
  2. Visual Studio Code 화면 왼쪽 사이드 바(Side Bar) 영역에 확장 기능 관리 창이 열리면, 상단 검색 창에서 "open in browser"로 검색합니다.
  3. 검색 창 아래에 "open in browser" 확장 기능이 표시되면 클릭합니다.
  4. 편집기 영역에 "open in browser" 확장 기능의 설명이 표시되면, "Install" 버튼을 클릭해 확장 기능을 설치합니다.

Tip

Visual Studio Code에서는 확장 기능(extensions)을 통해 Visual Studio Code의 기본 기능 외에도 많은 기능을 사용할 수 있으니 한번쯤 둘러보시기 바랍니다.

새로운 HTML 파일 생성

새로운 HTML 문서를 작성하기 위해 Visual Studio Code에서 새로운 파일을 만듭니다.

  1. Visual Studio Code 화면 왼쪽에 있는 작업 막대(Activity Bar)에서 탐색기(Explorer) 버튼을 클릭합니다.
  2. Visual Studio Code 화면 왼쪽 사이드 바(Side Bar) 영역에 탐색기 창이 열리면, 아래쪽 작업 디렉터리(workspace)의 파일이 표시되는 영역에 마우스 포인터를 가져가서 위쪽에 표시되는 "New File" 버튼을 클릭합니다.
  3. 그 아래에 새로운 파일 항목이 추가되면, 파일 이름을 hello.html이라고 입력합니다.

Tip

앞에서 말한 것처럼, 폴더나 파일의 이름에는 한글이나 빈칸을 쓰지 않고, 영문자와 숫자, 하이픈(-), 밑줄 문자(_)로만 작성하는 게 좋습니다.

새로운 파일을 만들면, 아래 그림처럼 Visual Studio Code 화면 오른쪽, 편집기 영역에 새로 만든 파일이 열립니다. 편집기 영역의 탭에는 열려 있는 파일의 이름이 표시됩니다. 파일을 수정하면 탭의 닫기 버튼이 동그랗게 원으로 표시됩니다. 파일을 수정하면 단축키 Ctrl+S를 눌러 저장합니다.

Tip

Visual Studio Code에서는 다양한 파일을 편집할 수 있고, 파일 종류(Language Mode; 언어 모드)에 따라 지원하는 기능도 달라집니다. 그래서 현재 편집하고 있는 파일의 언어 모드를 제대로 지정해야 합니다.

Visual Studio Code에서는 파일 확장자를 확인해 언어 모드를 결정합니다. HTML 파일의 파일 확장자는 .html이니, 파일 확장자를 제대로 지정했다면 Visual Studio Code는 현재 편집기 영역에 열린 파일이 HTML 파일이라고 인식합니다. 언어 모드는 Visual Studio Code 화면의 오른쪽 아래에 표시됩니다. 이 영역을 클릭하면 편집하고 있는 파일의 언어 모드를 바꿀 수 있습니다.

HTML 문서 편집

새로 만든 HTML 파일에 아래 내용을 입력합니다. 이때, 아래와 같은 사항을 주의해야 합니다.

  1. 대문자와 소문자를 구분해서 입력해야 합니다. 컴퓨터 프로그래밍에서는 대문자와 소문자를 다른 문자로 인식합니다.
  2. 오타가 나지 않도록 주의합니다.
  3. HTML 문서에서 사용하는 기호들은 정해진 의미가 있으므로 반드시 규칙대로 사용해야 합니다.
  4. 줄 앞에 빈칸(space) 여러 개를 넣는 것을 들여쓰기(indentation)라고 합니다. HTML 문서에서 들여쓰기는 HTML 요소들의 중첩을 표현합니다. Visual Studio Code에서는 탭(Tab) 키나 Ctrl+[ 키를 눌러 들여쓰기를 하며, 기본 설정으로는 빈칸 네 개가 들어갑니다.
  5. 복사해서 붙여넣기 하거나 자동 완성 기능을 사용하지 말고, 직접 입력합니다. 앞으로 수많은 HTML 문서를 작성하게 될 테니, 손에 익을 수 있도록 익숙해질 때까지는 반복해서 작성해야 합니다.
<!DOCTYPE html>
<html lang="ko">
    <head>
        <meta charset="UTF-8">
        <title>Hello World!</title>
    </head>
    <body>

        <h1>Hello World!</h1>

    </body>
</html>

Tip

웹 브라우저는 HTML 문서를 해석할 때, HTML 요소 사이에 있는 공백 문자(white spaces)는 무시합니다. 또한 내부 텍스트에 있는 공백 문자 여러 개는 빈칸 하나로 해석합니다. 따라서 HTML 문서를 작성하면서 줄 바꿈을 하거나 들여쓰기를 하는 건 웹 브라우저에게는 의미가 없습니다. 하지만 사람인 개발자는 다릅니다. HTML 요소들의 중첩된 관계를 파악하기 쉽도록 적절하게 줄 바꿈을 하고 들여쓰기를 하는 건 아주 중요합니다.

Tip

공백 문자(White spaces)는 편집기에서 따로 설정하지 않으면 보이지 않는 글자들을 말합니다. 빈칸(space)이나 엔터 키를 누르면 입력되는 개행 문자(new line), 탭 키를 누르면 입력되는 탭 문자(tab) 등이 공백 문자입니다.

Google Chrome으로 작성한 HTML 문서 확인

편집기로 작성한 HTML 문서는 제대로 작성했는지 웹 브라우저로 확인해봐야 합니다. 파일 탐색기를 열어 HTML 파일을 웹 브라우저로 열어 확인해야 하지만, 그렇게 하는 건 번거로우니 앞에서 설치한 "open in browser" 확장 기능을 사용해 Visual Studio Code에서 바로 웹 브라우저로 확인합니다.

Warning

HTML 문서를 수정하고 나서 웹 브라우저로 확인하기 전에 반드시 저장해야 합니다. 파일을 저장하지 않으면 웹 브라우저에서 수정한 내용을 확인할 수 없습니다.

Visual Studio Code의 편집기 영역에서 마우스 오른쪽 버튼을 클릭해 컨텍스트 메뉴를 엽니다. 컨텍스트 메뉴의 아래쪽에 "Open In Default Browser" 메뉴 항목과 "Open In Other Browsers" 메뉴 항목이 나오는지 확인합니다.

  • Google Chrome이 기본 웹 브라우저로 설정돼 있으면, "Open In Default Browser" 메뉴 항목을 선택하면 방금 작성한 HTML 문서를 Google Chrome으로 확인할 수 있습니다. 이 기능의 단축키는 Alt+B입니다.
  • Google Chrome이 기본 웹 브라우저가 아니면 "Open In Other Browsers" 메뉴 항목을 선택합니다. 그러면 화면 상단에 웹 브라우저 목록이 나오는데, 거기에서 "Google Chrome"을 선택해 방금 작성한 HTML 문서를 Google Chrome으로 확인합니다.

Tip

각 운영 체제의 기본 웹 브라우저를 바꾸는 방법은 구글이나 네이버에서 검색해보면 나옵니다.

방금 작성한 hello.html 파일을 웹 브라우저로 확인하면 위 그림과 같이 나옵니다.
