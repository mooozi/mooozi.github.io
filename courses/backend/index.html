프로그램 설치 디렉터리와 작업 디렉터리 생성

백 엔드 개발 관련 프로그램 설치 디렉터리 생성

백 엔드 애플리케이션을 개발하는 데 필요한 프로그램과 라이브러리를 한 곳에 모아두면 관리하기 편합니다.

Microsoft Windows에서는 파일 탐색기(Win+E 단축키)를 실행해, 위 그림과 같이 C 드라이브에 dev 폴더를 생성합니다. 이 폴더에 필요한 프로그램과 라이브러리를 저장할 겁니다. 폴더 이름에는 한글이나 빈칸을 넣지 않도록 합니다.

폴더설명
C:\dev\downloads다운로드한 파일들을 저장할 폴더
C:\dev\jdkJDK(Java Development Kit)를 설치할 폴더
C:\dev\intellijIDE인 IntelliJ IDEA를 설치할 폴더
C:\dev\tomcat아파치 톰캣(Apache Tomcat; WAS)을 설치할 폴더
C:\dev\databaseDBMS와 DB 관련 프로그램을 설치할 폴더
C:\dev\lib자바 라이브러리를 저장할 폴더

C:\dev 폴더에는 위 표에 나온 것처럼 JDK와 IDE 등을 설치하고 저장할 폴더를 만듭니다.

작업 디렉터리 생성

작업 디렉터리(Workspace)는 우리가 작성한 자바 소스 파일 등을 저장하는 디렉터리(directory)를 말합니다. 작업 영역 또는 작업 폴더라고 하기도 합니다.

자바 프로젝트는 작업 디렉터리 안에 만듭니다. 자바 프로젝트에는 패키지들이 들어가고, 패키지에 자바 클래스를 만듭니다.

Microsoft Windows에서는 파일 탐색기를 실행해, D 드라이브나 C 드라이브에 workspace 폴더를 만듭니다. 그리고 그 안에 backend 폴더를 만듭니다. 그러니까 작업 디렉터리의 경로는 D:\workspace\backend 또는 C:\workspace\backend가 됩니다.

작업 디렉터리는 시스템 드라이브(C 드라이브)가 아닌, 다른 드라이브에 만드는 것이 좋습니다. 컴퓨터에 C 드라이브만 있다면 어쩔 수 없겠지만, C 드라이브 외에 다른 드라이브가 있다면 그 드라이브에 작업 디렉터리를 만듭니다. 비대면으로 수업을 듣는 수강생은 USB 메모리나 외장 HDD 등을 가져와서, 거기에 작업 디렉터리를 만들도록 합니다.

Tip

폴더나 파일의 이름은 한글이나 빈칸을 쓰지 않고, 영문자와 숫자, 하이픈(-), 밑줄 문자(_)로만 작성하는 게 좋습니다.

사용하는 컴퓨터의 운영 체제가 Microsoft Windows라면, 기본적으로 파일 확장자가 보이지 않도록 설정되어 있으니 파일 확장자가 보이도록 설정하는 게 좋습니다.

아래 그림과 같이 파일 탐색기의 "보기" 메뉴에서 "파일 확장명" 항목을 체크하면 파일 확장자가 표시됩니다.

Tip

파일 확장자(Filename extension)는 파일의 이름에서 파일의 종류와 그 역할을 표시하기 위해 사용하는 부분입니다. 간단하게 확장자라고도 합니다. 대부분의 운영 체제에서는 파일 이름에서 마지막 점(.) 뒤에 나오는 부분을 확장자로 인식합니다.

자바 소스 파일의 파일 확장자는 .java입니다.

수업 중에 사용하는 프로그램

백 엔드 개발 과정을 진행하면서 여러 프로그램들을 사용하지만, 기본이 되는 프로그램은 아래 두 가지입니다.

  • Adoptium Eclipse Temurin JDK
  • IntelliJ IDEA Communication Edition

다른 프로그램들은 필요할 때 설치합니다.

Adoptium Eclipse Temurin JDK

자바 프로그램을 작성하고 실행하기 위해서는 JDK(Java Development Kit)가 필요합니다. 수업 시간에는 OpenJDK 중 하나인 Adoptium Eclipse Temurin JDK를 사용합니다.

Temurin JDK 설치 프로그램 다운로드

웹 브라우저로 Adoptium 웹사이트에 접속해서, 화면 왼쪽에 보이는 "Latest LTS Release" 버튼을 클릭해 사용하고 있는 운영 체제에 맞게 Temurin JDK 최신 버전을 다운로드해서 C:\dev\downloads 폴더에 저장합니다.

Temurin JDK 설치

다운로드한 설치 프로그램을 실행해서 Adoptium Eclipse Temurin JDK를 설치합니다.

"Custom Setup" 화면에서 JDK 설치 경로를 바꾸기 위해 "Browse" 버튼을 누릅니다.

새 설치 경로를 설정하기 위해 jdk-21.0.4.7-hotspot과 같은 부분을 선택한 다음, Ctrl+C 단축키를 눌러 클립보드에 복사합니다.

  1. 탐색 창 위쪽에 있는 "Look in" 영역에서, 이전에 만들어두었던 프로그램 설치 디렉터리, C:\dev\jdk 폴더를 선택합니다.
  2. 아래에 있는 "Folder name" 영역에 조금 전에 클립보드에 복사해두었던 내용을 덧붙여서 설치 경로를 C:\dev\jdk\jdk-21.0.4.7-hotspot\과 같이 지정합니다.
  3. "OK" 버튼을 눌러 설치 경로를 설정합니다.

다시 "Custom Setup" 화면으로 되돌아와서 JAVA_HOME 환경 변수를 추가하기 위해 아래와 같이 설정합니다.

  1. "Set or override JAVA_HOME variable" 항목 왼쪽에 있는 버튼을 누릅니다.
  2. 메뉴가 표시되면 "Will be installed on local hard drive" 항목을 선택합니다.
  3. "Next" 버튼을 눌러 설치를 계속 진행합니다.

설정을 마치고 "Install" 버튼을 눌러 Adoptium Eclipse Temurin JDK를 설치합니다.

Temurin JDK 설치 후 확인

Adoptium Eclipse Temurin JDK가 제대로 설치됐는지 확인하기 위해 Win+R 단축키를 눌러 실행 창을 엽니다.

위 그림과 같이 실행 창에서 cmd를 입력해 명령 프롬프트를 실행합니다.

명령 프롬프트가 실행되면 javac --version 명령을 입력하고 엔터 키를 누릅니다. 설치한 Temurin JDK의 버전 정보가 출력되면 제대로 설치된 겁니다.

IntelliJ IDEA Community Edition

자바 프로그램을 작성하기 위해 편집기가 필요합니다. 수업 시간에는 JetBrains 사에서 만든 IntelliJ IDEA Community Edition이라는 IDE(Integrated Development Environment; 통합 개발 환경)를 사용합니다.

IntelliJ IDEA Community Edition 설치 프로그램 다운로드

웹 브라우저로 IntelliJ IDEA 웹사이트에 접속해서, 화면 위쪽 우측에 보이는 "다운로드" 버튼을 클릭해, 다운로드 페이지로 이동합니다.

다운로드 페이지에서 화면을 조금 내려 무료 버전인 "Community Edition"을 다운로드해서 C:\dev\downloads 폴더에 저장합니다.

IntelliJ IDEA Community Edition 설치

IntelliJ IDEA Community Edition 설치 프로그램을 다운로드하면 실행합니다.

IntelliJ IDEA Community Edition의 설치 경로를 바꾸기 위해 "찾아보기" 버튼을 누릅니다.

이전에 만들어두었던 프로그램 설치 디렉터리, C:\dev\intellij 폴더에 IntelliJ IDEA Community Edition을 설치합니다.

바탕 화면에 바로가기를 만들도록 체크하고, PATH 환경 변수에 IntelliJ IDEA Community Edition의 경로를 추가하도록 체크합니다.

설정이 끝나면 IntelliJ IDEA Community Edition을 설치합니다.

설치가 끝나면 "마침" 버튼을 눌러 설치 프로그램을 종료합니다.

개발 환경 설정

이제 IntelliJ IDEA Community Edition으로 자바 프로그램을 작성하고 실행하기 위해 개발 환경을 설정합니다. 개발 환경을 설정하는 순서는 아래와 같습니다.

  1. IntelliJ IDEA Community Edition 실행
  2. 자바 프로젝트 생성
  3. 자바 프로그램 작성
  4. 자바 프로그램 실행
  5. IntelliJ IDEA Community Edition 설정

IntelliJ IDEA Community Edition 실행

바탕 화면에 만들어진 바로 가기를 더블 클릭해서 IntelliJ IDEA Community Edition을 실행합니다.

불러올 설정이 따로 없으니 "Skip import" 버튼을 눌러 IntelliJ IDEA Community Edition을 실행합니다. 그럼 아래 그림처럼 IntelliJ IDEA Community Edition이 실행됩니다.

자바 프로젝트 생성

작업 디렉터리에 새 자바 프로젝트를 만듭니다. 자바 프로젝트는 자바 애플리케이션을 만들기 위한 단위입니다. 수업 시간에는 과정이 진행되는 동안 이 프로젝트 안에 자바 클래스들(Java classes)을 작성하게 됩니다.

IntelliJ IDEA Community Edition 화면에서 "New Project" 버튼을 눌러 새 프로젝트를 만듭니다.

새 프로젝트 생성 창이 표시되면, 왼쪽 영역의 "New Project" 항목 아래에 "Java" 항목이 선택됐는지 확인합니다. "Name" 항목에는 프로젝트 이름으로 java를 입력합니다.

프로젝트 저장 위치(Location)을 작업 디렉터리로 설정하기 위해 "Location" 항목 오른쪽에 있는 "Browse" 버튼을 누릅니다. 그러면 "Select project File Directory" 창이 표시되고, 만들어두었던 작업 디렉터리, D:\workspace\backend 폴더를 선택합니다. 작업 디렉터리를 선택하고, "OK" 버튼을 눌러 "Select project File Directory" 창을 닫습니다.

"Build system" 항목은 기본 값인 "IntelliJ"로 그대로 두고, "JDK" 항목은 설치했던 JDK가 선택되어 있는지 확인합니다. 그 아래에 있는 "Add sample code" 항목은 체크를 해제합니다. 설정이 끝나면 "Create" 버튼을 눌러 새 프로젝트를 만듭니다.

작업 디렉터리에 새 프로젝트를 만들면, 위 그림처럼 IntelliJ IDEA Community Edition에 새 프로젝트가 열립니다. 이때 기본적인 파일과 폴더는 IntelliJ IDEA Community Edition이 만들어줍니다.

  1. IntelliJ IDEA Community Edition에서 프로젝트를 처음 만들면 알림(notification)이 몇 가지 표시됩니다. IntelliJ IDEA Community Edition에서는 지나간 알림을 화면 위쪽 우측에 있는 종 모양의 "Notifications" 버튼을 눌러 확인할 수 있습니다.
  2. Microsoft Windows에서는 Microsoft Defender가 시스템을 실시간으로 보호하도록 설정되어 있습니다. 이런 경우에는 IntelliJ IDEA Community Edition의 성능이 저하될 수 있으므로 IntelliJ IDEA Community Edition을 Microsoft Defender의 예외 목록에 추가하는 게 좋습니다. "Automatically" 버튼을 눌러 IntelliJ IDEA Community Edition에서 자동으로 예외 목록에 추가하도록 합니다.
  3. IntelliJ IDEA Community Edition에서는 UI 언어를 한국어로 설정할 수 있습니다. 하지만 개발자는 영문 용어에 익숙해져야 합니다. 그러니 어지간하면 UI 언어를 영어로 사용하는 게 좋습니다. 수업 시간에는 UI 언어를 영어로 사용합니다.

자바 프로그램 작성

자바 프로그램을 작성한다는 것은 자바 클래스를 만든다는 말과 같습니다. 앞에서 말했듯 자바 클래스는 패키지(package)에 포함됩니다. 그래서 자바 클래스를 만들려면 먼저 패키지부터 만들어야 합니다. 패키지는 폴더로 만들어지며, src 폴더에 만듭니다.

  1. IntelliJ IDEA Community Edition 화면 왼쪽에 보이는 프로젝트 탐색기에서 src 폴더를 선택하고, 마우스 오른쪽 버튼을 눌러 콘텍스트 메뉴(context menu)를 엽니다.
  2. 콘텍스트 메뉴의 가장 위에 있는 "New" 항목을 선택합니다.
  3. 새로운 메뉴가 표시되면 "Package" 항목을 선택해 새 패키지를 추가합니다.

새 패키지의 이름을 지정하는 창이 표시되면 day01이라고 입력하고 엔터 키를 누릅니다.

자바 프로그램은 클래스로 만들어집니다. 그래서 새로 생성한 패키지에 새 자바 클래스를 추가합니다.

  1. IntelliJ IDEA Community Edition 화면 왼쪽에 보이는 프로젝트 탐색기에서 day01 패키지를 선택하고, 마우스 오른쪽 버튼을 눌러 콘텍스트 메뉴를 엽니다.
  2. 콘텍스트 메뉴의 가장 위에 있는 "New" 항목을 선택합니다.
  3. 새로운 메뉴가 표시되면 "Java Class" 항목을 선택해 새 자바 클래스를 추가합니다.

새 패키지의 이름을 지정하는 창이 표시되면 Hello라고 입력하고 엔터 키를 누릅니다.

새 자바 클래스를 추가하면, IntelliJ IDEA Community Edition 화면 오른쪽에 편집기 창이 열립니다. 새로운 자바 클래스를 추가하면 위 그림처럼 package 선언과 클래스 정의가 자동으로 추가됩니다. 여기에 아래에 나온 프로그램을 입력합니다.

package day01;

public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello World!");
    }
}

이때, 아래 사항들을 주의해야 합니다.

  1. 대문자와 소문자를 구분해서 입력해야 합니다. 컴퓨터 프로그래밍에서는 대문자와 소문자를 다른 문자로 인식합니다.
  2. 오타가 나지 않도록 주의합니다.
  3. 자바 프로그램에서 사용하는 기호들은 정해진 의미가 있으므로 반드시 규칙대로 사용해야 합니다.

파일을 수정하면 Ctrl+S 단축키를 눌러 저장합니다.

자바 프로그램 실행

IntelliJ IDEA Community Edition에서는 자바 프로그램을 작성하고 나서 바로 실행할 수 있습니다.

IntelliJ IDEA Community Edition 화면 위쪽 중간쯤에 삼각형 화살표 버튼을 누르거나 편집기 창 네 번째 줄 왼쪽에 있는 삼각형 화살표 버튼을 누릅니다. 아니면, Shift+F10 단축키를 누르면 자바 프로그램이 실행됩니다. 이 단축키는 앞으로 많이 사용할 테니 기억해두는 게 좋습니다.

자바 프로그램을 실행하면 편집기 창 아래에 실행 창이 표시되고, 실행 결과가 출력됩니다.

IntelliJ IDEA Community Edition 설정

IntelliJ IDEA Community Edition에서 몇 가지 내용을 설정하기 위해 설정 창을 엽니다.

IntelliJ IDEA Community Edition 화면 상단 왼쪽에 있는 "Main Menu" 버튼을 눌러 메인 메뉴를 표시합니다.

메인 메뉴가 표시되면, "File" 메뉴 항목에서 "Settings" 항목을 찾아 클릭합니다. 설정을 변경하거나 확인해야 하는 경우가 종종 있으니 단축키를 외워두면 편합니다.

설정 창 위쪽 좌측에는 검색 창이 있으니 설정할 내용을 알고 있으면 검색해서 쉽게 찾아갈 수 있습니다.

"Unused" 경고 해제

IntelliJ IDEA Community Edition은 스프링 프레임워크를 지원하지 않기 때문에, 3개월 차 과정에서 스프링 프레임워크를 사용하는 동안 "Unused" 경고(warning)가 많이 나옵니다. 무시할 수 없을 정도로 많이 나오니 이 경고는 설정에서 끄는 게 좋습니다.

  1. 설정 창 왼쪽, 설정 분류에서 "Editor" 항목을 선택합니다.
  2. 그 아래에 있는 "Inspections" 항목을 선택합니다.
  3. 설정 창 오른쪽, 상세 설정에 있는 "Profile" 항목에서 "Default" 항목을 선택합니다.
  1. 설정 창 오른쪽, 상세 설정에서 "Java" 항목을 선택합니다.
  2. "Java" 항목 아래에 있는 "Declaration redundancy" 항목을 선택합니다.
  1. 아래로 내려가면 "Unused declaration" 항목을 찾을 수 있는데, 이 항목의 체크를 해제합니다.
  2. 설정 창 아래쪽 우측에 있는 "Apply" 버튼을 눌러 변경한 설정을 적용합니다.

대소문자를 구분하지 않고 자동 완성 기능이 동작하도록 설정

IntelliJ IDEA Community Edition은 강력한 자동 완성 기능(code completion)을 제공합니다. 다만 대문자로 시작하는 식별자는 첫 글자를 대문자로 작성해야 자동 완성 기능이 동작합니다. 첫 글자를 소문자로 입력해도 자동 완성 기능이 동작하도록 아래와 같이 설정합니다.

  1. 설정 창 왼쪽, 설정 분류에서 "Editor" 항목을 선택하고, 그 아래에 있는 "General" 항목을 선택합니다.
  2. "General" 항목 아래에 표시된 "Code Completion" 항목을 선택합니다.
  3. 설정 창 오른쪽, 상세 설정 위쪽에 나온 "Match case" 항목의 체크를 해제합니다.
  4. 설정 창 아래쪽 우측에 있는 "Apply" 버튼을 눌러 변경한 설정을 적용합니다.

파일 인코딩 설정

웹 애플리케이션에는 기본 문자 인코딩으로 UTF-8을 사용합니다. 문자 인코딩이 제대로 설정되어 있지 않으면 한글이 제대로 표시되지 않습니다. 따라서 문자 인코딩은 반드시 UTF-8로 설정하도록 합니다.

  1. 설정 창 왼쪽, 설정 분류에서 "Editor" 항목을 선택합니다.
  2. 그 아래에 있는 "File Encodings" 항목을 선택합니다.
  3. 설정 창 오른쪽, 상세 설정 위쪽에 있는 "Global Encoding" 항목과 "Project Encoding" 항목에서 "UTF-8" 항목을 선택합니다.
  4. "Default encoding for properties files" 항목에서 "UTF-8" 항목을 선택하고, 그 아래에 있는 "Transparent native-to-ascii conversion" 항목을 체크합니다.
  5. 설정 창 아래쪽 우측에 있는 "Apply" 버튼을 눌러 변경한 설정을 적용합니다.

JVM의 문자 인코딩 설정

IntelliJ IDEA Community Edition도 자바로 작성된 애플리케이션입니다. 그래서 IntelliJ IDEA Community Edition은 JVM에서 실행되고, 우리가 작성한 프로그램도 JVM에서 실행됩니다. 만약 JVM의 문자 인코딩이 제대로 설정되어 있지 않으면, 프로그램에서 한글을 출력하면 제대로 표시되지 않을 수도 있습니다.

  1. IntelliJ IDEA Community Edition의 메인 메뉴에서 "Help" 항목을 선택합니다.
  2. 하위 메뉴 중에서 "Edit Custom VM Options" 항목을 선택합니다.

"Edit Custom VM Options" 메뉴 항목을 선택하면, 아래 그림처럼 편집기 창에 idea64.exe.vmoptions 파일이 열립니다.

이 파일의 내용을 아래와 같이 변경합니다.

-Xmx2048m
-Dfile.encoding=UTF-8
-Dconsole.encoding=UTF-8

파일을 수정하고 나서 Ctrl+S 단축키를 눌러 파일을 저장하고 편집기 창을 닫습니다.

JVM 설정을 변경하면, IntelliJ IDEA Community Edition의 캐시를 제거하고 다시 시작하는 게 좋습니다.

  1. IntelliJ IDEA Community Edition의 메인 메뉴에서 "File" 항목을 선택합니다.
  2. 하위 메뉴 중에서 "Invalidate Caches" 항목을 선택합니다.

"Invalidate Caches" 창이 표시되면, "Invalidate and Restart" 버튼을 눌러 캐시를 제거하고 IntelliJ IDEA Community Edition을 다시 시작합니다.
